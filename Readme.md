# Audio Preprocessing Pipeline

A modular and scalable audio preprocessing pipeline developed for the AI-Based Intelligent Awareness System. This repository prepares raw environmental audio recordings for machine learning by performing dataset inspection, preprocessing, duration standardization, renaming, metadata generation, and stratified dataset splitting.

The processed dataset generated by this repository is intended to be used by the **Sound Detection Model** repository for training and evaluation.

---

## Features

- Audio dataset inspection
- Audio preprocessing and normalization
- Audio duration standardization
- Consistent audio file renaming
- Automatic metadata generation
- Stratified train, validation, and test dataset splitting
- Modular project structure
- Scalable pipeline for custom environmental sound datasets

# Modules

## 1. Dataset Inspection

Analyzes the collected audio dataset and provides an overview of:

- Supported audio files
- Invalid or corrupted files
- Dataset statistics
- Class distribution
- Audio properties

Run:

```bash
python -m src.inspection.inspect_dataset
```

---

## 2. Audio Preprocessing

Preprocesses all supported audio recordings.

Typical operations include:

- Audio loading
- Channel conversion
- Sampling rate standardization
- Noise handling (if applicable)
- Audio formatting

Output:

```text
Processed Data/
```

Run:

```bash
python -m src.preprocessing.preprocess_dataset
```

---

## 3. Duration Standardization

Standardizes the duration of every audio recording to ensure consistency across the dataset.

Output:

```text
Duration Standardized Data/
```

Run:

```bash
python -m src.duration.standardize_duration
```

---

## 4. Audio Renaming

Renames every processed audio file into a consistent naming convention.

Example:

```text
baby_crying_001.wav
baby_crying_002.wav
glass_breaking_015.wav
```

Output:

```text
Renamed Data/
```

Run:

```bash
python -m src.renaming.rename_dataset
```

---

## 5. Metadata Generation

Automatically generates a metadata file containing information for every processed audio file.

The generated metadata includes:

- filename
- filepath
- label
- environment
- duration (in seconds)
- split

Run:

```bash
python -m src.metadata.generate_metadata
```

---

## Dataset Splitting

During metadata generation, the dataset is automatically divided into:

- Train
- Validation
- Test

using **stratified sampling** provided by **Scikit-learn**, ensuring that every class maintains approximately the same distribution across all splits.

Default split ratios:

- Train: **80%**
- Validation: **10%**
- Test: **10%**

The split information is stored in the `split` column of `metadata.csv`.

Audio files are **not duplicated or moved** into separate folders. The metadata file serves as the single source of truth for dataset partitioning.
